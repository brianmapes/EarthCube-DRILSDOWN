an EarthCube project

Frequently Asked Questions

1. What's the vision?

The idea of visualization for algorithm design (VisAD) is that human-data interaction in a Graphical User Interface (GUI) can be a valuable source of insight for the quest to objectively measure important space-time processes, relationships, and entities.

DRILSDOWN aims to link a powerful free GUI (The IDV, embodying the VisAD idea) to a popular coding space (JuPyter notebooks and  iPython), for big data in geospace (including the atmosphere and ocean where the time domain is also big and highly structured).

2. Who is DRILSDOWN intended for?

Researchers and students seeking to extract meaning from big geospatial datasets, including multiple and distributed datasets.

3. What does DRILSDOWN do?

DRILSDOWN facilitates the 3 main steps to realizing the VisAD vision: Select, Interact, Capture.

    1. Select space-time regions ("cases") where results from a candidate algorithm look "interesting" enough to merit a "drilling down".
    2. Interact with GUI (tied to a Notebook) to examine the algorithm's results and intermediate steps in light of raw data underlays.
    3. Capture findings, and templates for repeating the process (identically on other data cases, or in a next round of refinements).

4. How are those three steps (select, interact, capture) represented in software?

Step 1 (select) is often rooted in a user's own candidate codes, which define "interesting" cases. A case consists of a space and time range (bounding box) and a label.

Step 2 (interact) requires an IDV bundle with data and candidate algorithms ready to execute on them. Preparing this initial bundle is part of Step 3, since the cycle must be primed with an initial case study.

Step 3 (capture) encompassed several sub-activities.

3a. IDV bundles are captured to seed Step 2, for repeating on other cases, or for the next round of refinement.

3b. Notebooks with case metadata, candidate algorithm information, computation calls, and screen captures from the interactions in Step 2 are captured as a scientific record. These case study notebooks can be published to a RAMADDA repository as a coherent set of JuPyter and IDV files, where they are viewable and downloadable as a replicable scientific experience.  

3c. A notebook like 3b. can be stripped down to its basics and captured as a seed, for repeating identically on other cases, of for the next round of refinement.

Each of these captured items has associated services, so Step 3 is a big composite item.

4. Is DRILSDOWN one thing, or several?

Based on the above, DRILSDOWN must be at least 3 things! It is better to view DRILSDOWN as a set of connectors that link JuPyter notebooks (used for steps 1 and 3), The IDV (for step 2), and an IDV-related repository-with-services called RAMADDA (for step 3).

These connectors include internal enhancements and plugins to The IDV and RAMADDA software packages, plus a drilsdown.py file and/or Jython shell for JuPyter that bring capabilities into the JuPyter notebook environment. 

Select (step 1) often consists of users' own candidate codes, which define "interesting" cases. For this reason, our development focuses on designing an interface for users to interact with step 2 (the core IDV GUI process). Our own scientific use cases are offered as models.

5. What parts are ready to use now?

We don't want to frustrate or disappoint the potential user base with premature releases that don't actually serve top-level VisAD goals. However, test versions that embody workflows from our atmosphere and ocean research are available. Volunteer "alpha testers" to extend and generalize these use cases will be recruited when a critical mass of functionality has been assembled and tested.

6. How do I install DRILSDOWN?

(DRAFT!) Here is a minimal checklist of installs tested recently for the CHAD_G5NR prototype use case:

    1. Install conda and JuPyter notebooks, by installing Anaconda (python2, not python3).

    2. In a terminal, install a few necessary packages
            conda install netcdf4 hdf4 pyqt=4.11 ipywidgets

    3. Install The IDV and 2 small plugin files as described in the Quickstart directions here.

    4. Download drilsdown.py from this GitHub page into a home directory folder called ~/ ipython/extensions.

    5. Install JyIDV from GitHub, culimating in the command python setup.py install

There is redundant functionality between 4. and 5. but it may be useful to have the entire set of related tools.


7. How do I use DRILSDOWN?

To activate the iPython extension within a JuPyter notebook, you execute a cell containing 

%reload_ext drilsdown

Seed notebooks for Step 2 can contain this line as well as other cues to the user at execution time.
